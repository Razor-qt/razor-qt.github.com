Detailed Description

The XdgMenu class implements the "XDG Base Directory Specification" from freedesktop.org. This specification defines where these files should be looked for by defining one or more base directories relative to which files should be located.

See also:
http://standards.freedesktop.org/basedir-spec/basedir-spec-latest.html

QString XdgDirs::cacheHome ( bool  createDir = true) [static]

Returns the path to the directory that corresponds to the $XDG_CACHE_HOME. If createDir is true, the function will create the directory.

$XDG_CACHE_HOME defines the base directory relative to which user specific non-essential data files should be stored. If $XDG_CACHE_HOME is either not set or empty, a default equal to $HOME/.cache should be used.

QStringList XdgDirs::configDirs ( ) [static]

Returns a list of all directories that corresponds to the $XDG_CONFIG_DIRS. $XDG_CONFIG_DIRS defines the preference-ordered set of base directories to search for configuration files in addition to the $XDG_CONFIG_HOME base directory. If $XDG_CONFIG_DIRS is either not set or empty, a value equal to /etc/xdg should be used.

QString XdgDirs::runtimeDir ( ) [static]

Returns the path to the directory that corresponds to the $XDG_RUNTIME_DIR. $XDG_RUNTIME_DIR defines the base directory relative to which user-specific non-essential runtime files and other file objects (such as sockets, named pipes, ...) should be stored. The directory MUST be owned by the user, and he MUST be the only one having read and write access to it. Its Unix access mode MUST be 0700.
